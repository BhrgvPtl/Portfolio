Retail — Supply Chain Optimization

Inventory Demand Forecasting Platform

Delivered a demand forecasting program for 420+ SKUs that powers weekly replenishment and merchandising decisions across 60 retail locations.

Role

Principal Data Scientist

Timeline

5 months

Impact

18% fewer stock-outs • $1.1M working capital reduction

Business Problem

Northwind Retail struggled with manual, spreadsheet-based replenishment for seasonal inventory. Regional managers spent days reconciling supplier orders and still experienced chronic stock-outs for high-velocity items while over-ordering slow movers.

Approach

  1. Centralized historical POS data, promotions, and supplier lead times in BigQuery.
  2. Built a feature pipeline in SQL and dbt to capture lagged sales, events, and weather.
  3. Evaluated Prophet, ARIMA, and gradient boosting models in a rolling-origin backtest.
  4. Containerized the champion model and scheduled batch forecasts using Cloud Composer.
  5. Published forecast accuracy scorecards in Tableau for regional demand planners.

Tools & Methods

  • BigQuery
  • dbt
  • Python & Prophet
  • Cloud Composer
  • Looker & Tableau
  • GitHub Actions

Data Story

Combining transaction logs with local events uncovered why traditional models underperformed during holiday periods. Scenario analysis quantified the ROI of right-sizing inventory, and leadership used interactive dashboards to drill into store-level forecasts. We emphasized the narrative around customer experience: shelves staying stocked for loyalty program members.

Results

  • Reduced stock-outs across key categories by 18% during the first holiday season.
  • Improved forecast MAPE by 27% compared with legacy spreadsheets.
  • Decreased working capital tied up in safety stock by $1.1M.
  • Delivered store-level simulation tools that guided supplier negotiations.

Productionization

Forecast jobs are orchestrated in Cloud Composer with environment-specific configuration stored in Terraform. Automated data quality checks run via Great Expectations, and CI/CD pipelines enforce SQL linting, unit tests, and deployment approvals before changes promote to production.
